## 9. Trainer-Specific Pricing (Per Resource)

Requirement change:

- trainers have different prices

Changes made:

- schema changed to store per-instructor pricing by period
- seed updated with multiple trainers and distinct prices
- booking backend pricing now uses selected trainer’s price (override)
- availability for training filters trainers by sport
- booking UI requires trainer selection and shows trainer-dependent total
- admin instructors page supports inline editing of trainer prices

## 10. Hour-Based Timeslot Enforcement

Requirement change:

- only hour-based slots allowed (`09:00-10:00`), no `09:15-10:15`

Changes made:

- availability engine now generates start times aligned to hour only
- booking validation requires `startTime` minute component `00`
- DB booking path also enforces hour-only start
- availability API metadata updated (`slotGranularityMin=60`)

## 11. Booking Consistency / “Slot still available after booking” Fix

Observed issue:

- user booked a slot, refreshed page, slot still appeared available

Likely root cause:

- booking API could silently fall back to demo mode on DB errors, returning success-like payload without DB persistence

Fixes:

- disabled silent demo fallback by default (`ALLOW_DEMO_FALLBACK=false`)
- booking API now returns real errors when DB path fails (unless fallback explicitly enabled)
- availability API also avoids silently masking DB failures by default
- booking UI now displays demo-fallback warning if fallback is explicitly enabled and used

## 12. Booking Authorization Rule Update

New rule implemented:

- court rental bookings require authenticated registered user

Enforced in:

- booking UI (`/book`) - warning + register/login links
- `POST /api/bookings` - returns `401` if court rental is attempted without session

Training bookings remain possible without account unless further policy changes are requested.

## Important Local Actions Required After Latest Schema Changes

Because `Instructor` schema changed (sport + per-period prices), local DB/schema must be updated:

1. Stop `next dev` (to avoid Prisma Windows file-lock issues)
2. Run migration:
   - `npx prisma migrate dev --name trainer_pricing_and_booking_auth`
3. Regenerate Prisma Client:
   - `npx prisma generate`
4. Reseed:
   - `npm run db:seed`

If `npx prisma generate` fails with `EPERM` on Windows:

- ensure no running Node/Next/Prisma process is locking `node_modules/.prisma/client/*`
- retry
- temporary fallback for types only:
  - `npx prisma generate --no-engine`
