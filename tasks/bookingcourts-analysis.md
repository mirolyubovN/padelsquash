# Анализ bookingcourts.ru/courts (2026-03-05)

## Что удалось получить из страницы

- Доступный HTML-фрагмент предельно простой: заголовок и сетка иконок-клубов.
- На странице нет длинных описаний, сложных блоков статистики, фильтров и вторичных действий.
- Паттерн взаимодействия: один главный выбор на экран -> переход дальше.

## Принципы, которые берем в редизайн

- Один главный сценарий на экран, без лишних панелей.
- Крупные карточки-выборы вместо плотных таблиц там, где это возможно.
- Минимум финансовых и технических деталей для операционных ролей.
- Роли должны видеть только то, что нужно им по задаче.

## Что внедрено в этом шаге

- Введены роли `super_admin`, `admin`, `trainer`.
- Финансово-чувствительные разделы (цены/спорты) ограничены только `super_admin`.
- Для обычного `admin` скрыты элементы выручки в дашборде и суммовые поля в бронированиях.
- Добавлен отдельный упрощенный маршрут `/trainer/schedule` для самостоятельного управления расписанием тренера.
