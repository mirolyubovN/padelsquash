## Design Decisions

- Currency and sample pricing are in `KZT` only.
- Timezone helper is generic (`src/lib/time/venue-timezone.ts`) with default `Asia/Almaty`.
- Availability and pricing APIs currently use demo datasets so UI/API contracts can progress before DB wiring.
- Concurrency approach is isolated in `src/lib/bookings/concurrency.ts` to make DB integration straightforward.

## Update (Requirement Change Applied)

- Booking sessions simplified to fixed `60` minutes only.
- Buffer between sessions removed from app logic/demo behavior (services now seeded/demoed with `bufferMin=0`).
- Pricing simplified from rule engine usage to fixed matrix:
  - sport (`padel` / `squash`)
  - component (`court` / `instructor`)
  - period (`morning` / `day` / `evening_weekend`)
- Training session price now equals `court price + instructor price` for the selected sport/period.
- Demo/hardcoded data consolidated in `src/lib/demo/hardcoded-data.ts`.
